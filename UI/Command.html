概述

Command类是向菜单和Ruby工具栏添加工具的首选类。例如，您可以添加一个菜单项并直接将其传递给代码块，或者您可以先创建一个Command。

使用Command可以更好地控制项目在UI中的工作方式，并且允许UI中的多个位置调用相同的代码。例如，您可能希望工具栏按钮和上下文单击菜单项都指向同一命令，并从代码中的单个位置控制工具提示及其“灰化”。

例子：

# 您可以添加菜单项作为程序块，如这里所示，但是您无法控制它是否为灰色。
UI.menu("Draw").add_item("My Procedure") {
  UI.messagebox("My Procedure")
}

# 最好先创建一个command对象。
cmd = UI::Command.new("Tester") {
  UI.messagebox("My Command")
}
cmd.menu_text = "My Command"
cmd.set_validation_proc {
  if Sketchup.active_model.selection.length == 0
    MF_GRAYED
  else
    MF_ENABLED
  end
}
UI.menu("Draw").add_item cmd

版本：

  • SketchUp 6.0

↑ #large_icon ⇒ String

large_icon方法返回命令的大图标的图标文件。

例子：

# This command displays Hello World on the screen when clicked

cmd.small_icon = "ToolPencilSmall.png"
cmd.large_icon = "ToolPencilLarge.png"

↑ #large_icon=(path) ⇒ String

large_icon = method 用于标识命令的大图标的图标文件。大图标应为24x24像素图像，以获得最佳显示质量。

从SketchUp 2016开始，可以为光标提供矢量图像。适用于Windows的SVG格式和适用于OS X的PDF格式。由于矢量图像适用于小图标和大图标尺寸，因此您可以选择仅对两种变体使用一个矢量图像。

例子：

toolbar = UI::Toolbar.new "Test"
# This command displays Hello World on the screen when clicked
cmd = UI::Command.new("Test") { UI.messagebox("Hello World") }
cmd.small_icon = "ToolPencilSmall.png"
cmd.large_icon = "ToolPencilLarge.png"
toolbar = toolbar.add_item cmd
toolbar.show

参数:

  • path (String) —

    大图标的路径。

  • (String)

↑ #set_validation_proc { ... } ⇒ UI::Command

该 #set_validation_proc 方法允许您更改基于用户状态的命令是否被启用，检查等。例如，除非用户具有当前选择，否则您可能希望禁用您的命令。

例子：

# Create a command object.
cmd = UI::Command.new("Tester") {
  UI.messagebox("My Command")
}
cmd.menu_text = "My Command"
cmd.set_validation_proc {
  if Sketchup.active_model.selection.length == 0
    MF_GRAYED
  else
    MF_ENABLED
  end
}
UI.menu("Draw").add_item(cmd)

Yields:

Yield 返回:

  • (Integer) —

    MF_ENABLED, MF_DISABLED, MF_CHECKED, MF_UNCHECKED, or MF_GRAYED

返回:

  • (UI::Command)
