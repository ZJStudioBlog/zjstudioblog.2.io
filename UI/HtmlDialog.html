概述

Ruby HtmlDialog类允许您创建Ruby中的HTML对话框并与之交互。这是在SketchUp中生成复杂的嵌入式UI的最佳方法，但它通常需要HTML和JavaScript专业知识。

如果您的目标是简单地向用户显示网站，请考虑使用 #openURL，它会在默认浏览器中向他们显示网页，而不是在SketchUp中的对话框中。

↑ #initialize(properties) ⇒ HtmlDialog

该new方法用于创建一个新的HtmlDialog。

所述 properties hash接受一个可选的键style ，其中值为：

UI::HtmlDialog::STYLE_DIALOG

HtmlDialog保持在SketchUp的顶部。

UI::HtmlDialog::STYLE_WINDOW

HtmlDialog可以在SketchUp后面，并且当SketchUp失去焦点时不会消失。

UI::HtmlDialog::STYLE_UTILITY

HtmlDialog显示为小标题栏并保持在SketchUp之上。

例子:

dialog = UI::HtmlDialog.new(
{
  :dialog_title => "Dialog Example",
  :preferences_key => "com.sample.plugin",
  :scrollable => true,
  :resizable => true,
  :width => 600,
  :height => 400,
  :left => 100,
  :top => 100,
  :min_width => 50,
  :min_height => 50,
  :max_width =>1000,
  :max_height => 1000,
  :style => UI::HtmlDialog::STYLE_DIALOG
})
dialog.set_url("http://www.sketchup.com")
dialog.show

实例方法详细信息

↑ #add_action_callback(callback_name) {|action_context, ...| ... } ⇒ Boolean

注意:

当关闭HtmlDialog时，将清除该实例的所有回调。如果需要再次打开对话框，请重新附加它们。

该 #add_action_callback 法建立你的HTML对话框可以调用执行某些功能的Ruby的回调方法。

使用 sketchup.callback_method_name 从html对话框中调用回调方法。html对话框中的JavaScript将使用相同数量的参数调用回调。

该调用是异步的。在调用Ruby回调之前，JavaScript调用可能会返回。使用 onCompleted 回调来获得完成通知。

基本类型（如布尔型，整型，字符串型，数组和hashes型）在Ruby和JavaScript之间自动转换。

例子:

Ruby Code

dialog.add_action_callback("say") { |action_context, param1, param2|
  puts "JavaScript said #{param1} and #{param2}"
}

JavaScript

sketchup.say('Hello World', 42);

JavaScript with callback

sketchup.say('Hello World', 42, {
  onCompleted: function() {
    console.log('Ruby side done.');
  }
});

Returns 如果动作成功添加，则为true，否则为false。

参数:

  • callback_name (字符串型) —

    从html对话框调用的回调方法的名称。

Yields:

  • (action_context, ...)

Yield 参数:

  • action_context (Object) —

    action_context当前未使用。

  • ... (Object) —

    从JavaScript发送的参数。

返回值:

  • (布尔型) —

    如果动作成功添加，则为true，否则为false。
