概述

SketchupExtension类包含允许您创建和操作SketchUp扩展的方法。扩展是可以使用“扩展程序管理器”（“首选项”对话框的“扩展”面板）加载和卸载的Ruby脚本。通常，您应该将ruby脚本注册为扩展，以便SketchUp用户能够通过用户界面禁用它。

这里的想法是采用实际创建您的功能的ruby脚本并将其放在/ Plugins文件夹之外的某个文件夹中，最常见的是像/ Plugins / MyExtension这样的子目录。然后在/ Plugins目录中创建一个新的ruby脚本，该脚本将设置扩展条目并在用户打开扩展时加载原始脚本。

下面是一个示例扩展加载脚本。对于本例，以下代码将保存在/Plugins/StairTools.rb中。而实际的插件本身会存在于/Plugins/StairTools/core.rb中。

您可以在/ Plugins /文件夹下找到SketchUp附带的两个示例扩展，su_dynamiccomponents.rb和su_sandboxtools.rb。

例子:

# 在扩展列表中创建一个条目，该条目加载一个名为core.rb的脚本。
require 'sketchup.rb'
require 'extensions.rb'

stair_extension = SketchupExtension.new('Stair Tools", "StairTools/core.rb')
stair_extension.version = '1.0'
stair_extension.description = 'Tools to draw stairs automatically.'
Sketchup.register_extension(stair_extension, true)

版本：

  • SketchUp 6.0

构造函数详细信息

↑ #initialize(title, path) ⇒ Sketchup::Extension

new方法用于创建一个新的SketchupExtension对象。一旦创建了扩展对象，它将不会出现在“扩展管理器”对话框中，直到您使用Sketchup.register_extension方法注册它。

例子:

# 在扩展列表中创建一个条目，该条目加载一个名为core.rb的脚本。
extension = SketchupExtension.new('Stair Tools', 'StairTools/core.rb')

# 然后一定要注册它。通过传递第二个参数true，您将告诉SketchUp在默认情况下加载扩展名。
Sketchup.register_extension(extension, true)

参数：

  • title (String) —

    扩展名

  • path (String) —

    加载插件的脚本的相对路径。

版本：

  • SketchUp 6.0

实例方法详细信息

↑ #check ⇒ Boolean

加载扩展名，意味着立即解释基础ruby脚本。这相当于在Preferences> Extensions列表中选中扩展名的复选框。

例子:

# 这将注册扩展，这是它出现在SketchUp的首选项>扩展列表中的必要步骤
ext_c = SketchupExtension.new('Stair Tools C', 'StairTools/core.rb')
Sketchup.register_extension(ext_c, false)

# 这将加载扩展。
ext_c.check

返回：

  • (Boolean) —

    加载是否成功

↑ #uncheck ⇒ Boolean

卸载扩展程序。这相当于取消选中Preferences> Extensions列表中的扩展名复选框。

请注意，从技术上讲，扩展不会在当前SketchUp会话期间停止运行的情况下“卸载”，但下次用户重新启动SketchUp时，扩展将不会处于活动状态。

例子:

# This unloads all extensions. The next time SketchUp starts, none of
# the extensions will be active.
Sketchup.extensions.each { |extension|
  extension.uncheck
}

返回：

  • (Boolean) —

    卸载是否成功

版本：

  • SketchUp 8.0 M2
